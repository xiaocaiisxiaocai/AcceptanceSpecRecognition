# 设计说明：Excel 导入（按列序号配置）

## 目标
以“用户明确指定列序号”的方式导入 Excel 工作表中的验收规格数据，覆盖 Excel 常见复杂场景：跨行表头、非第 1 列起始、合并单元格、混合数据类型与公式。

## 核心设计决策

### 1) 不复用列映射规则
Excel 导入**不使用**现有“列映射规则（按表头文本匹配）”来自动映射列。原因：
- Excel 表头经常跨行、合并、甚至存在多层级标题，文本匹配误判概率更高
- 用户需求明确偏好“指定第几列是什么”，可预测性更强

### 2) 配置模型（按工作表）
每个工作表一个配置项（建议服务端与前端统一 1-based）：
- `sheetIndex` / `sheetName`
- `headerRowStart`（表头起始行，1-based，可选）
- `headerRowCount`（表头行数，0 表示无表头，仅按列号导入）
- `dataStartRow`（数据起始行，1-based）
- `dataEndRow`（可选；为空表示自动到已用区域末尾）
- `projectColumn`（必填，1-based）
- `specificationColumn`（必填，1-based）
- `acceptanceColumn`（可选）
- `remarkColumn`（可选）

本变更不提供“连续空行提前停止”配置：读取范围以已用区域末尾为准。

UI 可允许用户输入列号数字，同时在旁边显示字母（A/B/C...）帮助确认。

### 3) 合并单元格展开策略
Excel 中合并单元格常用于：
- 表头分组（跨列/跨行）
- 数据区“同一项目多行”的视觉合并（跨行）

解析器在导入/预览前先构建“展开后”的逻辑网格：
- 对每个合并区域，取左上角单元格值，填充到区域内所有单元格
- 对缺失值的单元格读取时优先取展开后的值

### 4) 已用区域与结束条件
- 优先以工作表的“已用区域（Used Range）”作为最大边界，避免扫描整张表
- 本变更不做“连续空行提前停止”：读到已用区域末尾为止；空行在生成导入项时被跳过

### 5) 单元格取值与类型规整
需要统一转成字符串入库：
- 字符串：保留原文本，标准化空白（trim、全角空格等可选）
- 数字/日期：按显示文本或规范化格式（建议先按显示文本，避免精度/时区争议）
- 公式：本变更不考虑公式计算场景；按读取到的值/文本处理（不做公式重算与特别提示）
- 富文本/换行：保留换行（`\n`），由前端预览与后续匹配逻辑决定是否压缩

## 风险与边界
- 大文件内存：部分 Excel 库会一次性加载整个工作簿；需要设置上传大小上限与预览行列上限
- 公式计算：本变更不支持公式计算；若未来出现依赖公式结果的 Excel，将单独扩展
- 多表格区域：本变更不支持“一张表多块区域”；若未来出现，将引入“显式区域配置”

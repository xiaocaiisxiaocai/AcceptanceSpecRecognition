# Matching Engine Capability

## ADDED Requirements

### Requirement: 文本相似度匹配
系统必须（SHALL）支持基于文本相似度算法的匹配功能。

#### Scenario: Levenshtein距离匹配
- **GIVEN** 用户选择Levenshtein算法
- **AND** 输入查询文本"不锈钢管材"
- **WHEN** 系统执行匹配
- **THEN** 系统计算查询文本与数据库中所有【项目+规格】的Levenshtein距离
- **AND** 返回相似度得分（0-1之间，1为完全匹配）
- **AND** 按得分降序排列结果

#### Scenario: Jaccard相似度匹配
- **GIVEN** 用户选择Jaccard算法
- **AND** 输入查询文本"304不锈钢 外径50mm"
- **WHEN** 系统执行匹配
- **THEN** 系统将文本分词后计算词集合的Jaccard系数
- **AND** 返回相似度得分
- **AND** 按得分降序排列结果

#### Scenario: 余弦相似度匹配
- **GIVEN** 用户选择余弦相似度算法
- **AND** 输入查询文本
- **WHEN** 系统执行匹配
- **THEN** 系统将文本向量化（TF-IDF或词频）
- **AND** 计算向量间的余弦相似度
- **AND** 返回相似度得分

### Requirement: 多列组合查找
系统必须（SHALL）支持基于【项目】和【规格】两列组合进行查找匹配。

#### Scenario: 组合匹配
- **GIVEN** 用户输入项目="不锈钢管"，规格="Φ50×3mm"
- **WHEN** 系统执行匹配
- **THEN** 系统分别计算项目和规格的相似度
- **AND** 按权重（默认各50%）加权计算综合得分
- **AND** 返回综合得分最高的匹配结果

#### Scenario: 自定义权重
- **GIVEN** 用户设置项目权重70%，规格权重30%
- **WHEN** 系统执行匹配
- **THEN** 系统按自定义权重计算综合得分
- **AND** 得分计算公式：综合得分 = 项目得分×0.7 + 规格得分×0.3

### Requirement: Embedding向量匹配
系统必须（SHALL）支持基于Embedding向量的语义匹配功能。

#### Scenario: 生成查询向量
- **GIVEN** 用户输入查询文本"304不锈钢圆管 直径50"
- **WHEN** 系统准备匹配
- **THEN** 系统调用配置的Embedding模型
- **AND** 生成查询文本的向量表示

#### Scenario: 向量相似度计算
- **GIVEN** 系统已生成查询向量
- **AND** 数据库中已缓存所有数据的向量
- **WHEN** 系统执行匹配
- **THEN** 系统计算查询向量与所有缓存向量的余弦相似度
- **AND** 返回相似度得分（0-1之间）
- **AND** 按得分降序排列结果

#### Scenario: 批量向量预计算
- **GIVEN** 用户导入新数据后首次使用Embedding匹配
- **WHEN** 系统检测到数据无缓存向量
- **THEN** 系统批量调用Embedding模型生成向量
- **AND** 显示生成进度
- **AND** 支持取消操作
- **AND** 将向量存入缓存表

---

### Requirement: LLM增强匹配
系统必须（SHALL）支持LLM+Embedding的混合匹配模式。

#### Scenario: LLM语义增强
- **GIVEN** 用户选择LLM+Embedding匹配模式
- **AND** 输入查询文本"SUS304无缝钢管 50mm外径"
- **WHEN** 系统执行匹配
- **THEN** 系统首先使用Embedding获取Top-N候选结果
- **AND** 调用LLM对候选结果进行语义分析
- **AND** LLM返回重新排序的结果和匹配理由

#### Scenario: LLM匹配解释
- **GIVEN** LLM完成匹配分析
- **WHEN** 系统返回结果
- **THEN** 每个匹配结果包含LLM生成的解释
- **AND** 解释说明为什么认为该结果匹配
- **AND** 解释使用中文自然语言

### Requirement: 阈值过滤
系统必须（SHALL）支持可配置的匹配阈值，低于阈值的结果自动过滤。

#### Scenario: 应用阈值过滤
- **GIVEN** 用户设置匹配阈值为0.8
- **AND** 匹配结果中有得分0.95、0.82、0.75、0.60的四条记录
- **WHEN** 系统应用阈值过滤
- **THEN** 系统返回得分0.95和0.82的两条记录
- **AND** 得分0.75和0.60的记录被过滤

#### Scenario: 无结果达到阈值
- **GIVEN** 用户设置匹配阈值为0.9
- **AND** 所有匹配结果得分均低于0.9
- **WHEN** 系统应用阈值过滤
- **THEN** 系统返回空结果集
- **AND** 提示"未找到满足阈值的匹配结果"

#### Scenario: 阈值范围验证
- **GIVEN** 用户尝试设置阈值为1.5
- **WHEN** 系统验证输入
- **THEN** 系统显示错误"阈值必须在0到1之间"

---

### Requirement: 匹配结果取最高分
系统必须（SHALL）在多个匹配结果中自动选择得分最高的结果用于填充。

#### Scenario: 自动选择最高分
- **GIVEN** 匹配返回多个结果，得分分别为0.95、0.88、0.82
- **WHEN** 系统准备填充数据
- **THEN** 系统自动选择得分0.95的记录
- **AND** 使用该记录的【验收】和【备注】列进行填充

#### Scenario: 得分相同时的处理
- **GIVEN** 两条记录得分相同，均为0.90
- **WHEN** 系统选择结果
- **THEN** 系统选择更新时间较新的记录
- **AND** 在预览中标注"存在多个相同得分结果，已选择最新记录"

---

### Requirement: 得分计算详情展示
系统必须（SHALL）在预览界面展示详细的得分计算过程。

#### Scenario: 相似度匹配得分详情
- **GIVEN** 用户使用相似度匹配查看预览
- **WHEN** 系统显示匹配结果
- **THEN** 显示以下信息：
  - 使用的算法名称（如Levenshtein）
  - 查询文本【项目】和【规格】
  - 匹配文本【项目】和【规格】
  - 项目列相似度得分
  - 规格列相似度得分
  - 权重配置
  - 综合得分计算公式和结果
  - 阈值状态（达标显示绿色，未达标显示红色）

#### Scenario: Embedding匹配得分详情
- **GIVEN** 用户使用Embedding匹配查看预览
- **WHEN** 系统显示匹配结果
- **THEN** 显示以下信息：
  - 使用的Embedding模型名称
  - 向量维度
  - 余弦相似度得分
  - 阈值状态

#### Scenario: LLM+Embedding匹配得分详情
- **GIVEN** 用户使用LLM+Embedding匹配查看预览
- **WHEN** 系统显示匹配结果
- **THEN** 显示以下信息：
  - Embedding初筛得分
  - LLM重排序后得分
  - LLM匹配解释文本
  - 使用的LLM模型名称
  - 综合得分计算方式
  - 阈值状态

### Requirement: 已填充行跳过规则
系统必须（SHALL）在填充时自动跳过已有数据的行。

#### Scenario: 验收和备注都已填充
- **GIVEN** 目标Word表格某行的【验收】列为"OK 符合标准"
- **AND** 该行【备注】列为"已检验"
- **WHEN** 系统准备填充该行
- **THEN** 系统跳过该行
- **AND** 在预览中标记该行为"已跳过（已有数据）"
- **AND** 不覆盖原有内容

#### Scenario: 仅验收列已填充
- **GIVEN** 目标Word表格某行的【验收】列为"NG"
- **AND** 该行【备注】列为空
- **WHEN** 系统准备填充该行
- **THEN** 系统跳过该行
- **AND** 在预览中标记该行为"已跳过（验收已填）"

#### Scenario: 仅备注列已填充
- **GIVEN** 目标Word表格某行的【验收】列为空
- **AND** 该行【备注】列为"特殊要求"
- **WHEN** 系统准备填充该行
- **THEN** 系统跳过该行
- **AND** 在预览中标记该行为"已跳过（备注已填）"

#### Scenario: 验收和备注都为空
- **GIVEN** 目标Word表格某行的【验收】列为空
- **AND** 该行【备注】列为空
- **WHEN** 系统准备填充该行
- **THEN** 系统正常执行匹配和填充
- **AND** 在预览中显示匹配结果

#### Scenario: 空白字符判定
- **GIVEN** 目标Word表格某行的【验收】列仅包含空格或换行符
- **WHEN** 系统判断是否已填充
- **THEN** 系统将仅含空白字符的单元格视为空
- **AND** 正常执行匹配和填充

#### Scenario: 跳过统计
- **GIVEN** 批量填充完成
- **WHEN** 系统显示填充结果
- **THEN** 系统显示统计信息：
  - 总行数
  - 已跳过行数（已有数据）
  - 成功填充行数
  - 无匹配结果行数

### Requirement: 同义词增强匹配
系统必须（SHALL）支持使用同义词增强匹配准确性。

#### Scenario: 相似度匹配使用同义词
- **GIVEN** 用户启用同义词功能
- **AND** 存在同义词组"不锈钢,不鏽鋼,SUS,Stainless"
- **AND** 查询文本为"SUS304管材"
- **AND** 数据库中存在"不锈钢304管材"
- **WHEN** 系统执行相似度匹配
- **THEN** 系统识别"SUS"和"不锈钢"为同义词
- **AND** 在计算相似度时将同义词视为完全匹配
- **AND** 提高匹配得分

#### Scenario: Embedding匹配使用同义词
- **GIVEN** 用户启用同义词功能
- **AND** 存在同义词组"外径,OD,Outer Diameter"
- **AND** 查询文本为"OD50mm"
- **WHEN** 系统生成Embedding向量前
- **THEN** 系统将"OD"替换为标准词"外径"
- **AND** 使用"外径50mm"生成向量
- **AND** 提高与含"外径"数据的匹配度

#### Scenario: 得分详情显示同义词信息
- **GIVEN** 匹配过程中使用了同义词替换
- **WHEN** 系统显示得分详情
- **THEN** 显示同义词替换信息：
  - 原始文本
  - 替换后文本
  - 使用的同义词组

#### Scenario: 同义词功能禁用
- **GIVEN** 用户禁用同义词功能
- **WHEN** 系统执行匹配
- **THEN** 系统不进行同义词替换
- **AND** 使用原始文本进行匹配

---

### Requirement: 预处理集成
系统必须（SHALL）在匹配前应用文本预处理。

#### Scenario: 简繁转换后匹配
- **GIVEN** 用户启用简繁转换（简体→台湾繁体）
- **AND** 查询文本为"不锈钢"（简体）
- **AND** 数据库中存在"不鏽鋼"（繁体）
- **WHEN** 系统执行匹配
- **THEN** 系统先将两者统一转换为台湾繁体
- **AND** 使用转换后的文本计算相似度
- **AND** 成功匹配"不鏽鋼"

#### Scenario: 预处理顺序
- **GIVEN** 用户同时启用简繁转换和同义词功能
- **WHEN** 系统执行匹配
- **THEN** 系统先执行简繁转换
- **AND** 再执行同义词替换
- **AND** 最后进行匹配计算

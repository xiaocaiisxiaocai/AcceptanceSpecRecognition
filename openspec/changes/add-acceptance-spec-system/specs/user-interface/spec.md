# User Interface Capability

## ADDED Requirements

### Requirement: 主界面布局
系统必须（SHALL）提供清晰的WinForms主界面布局。

#### Scenario: 主窗体结构
- **GIVEN** 用户启动应用程序
- **WHEN** 主窗体加载完成
- **THEN** 界面显示以下区域：
  - 顶部：菜单栏和工具栏
  - 左侧：功能导航面板
  - 中间：主工作区域
  - 底部：状态栏（显示当前状态和进度）

#### Scenario: 功能导航
- **GIVEN** 用户在主界面
- **WHEN** 用户点击左侧导航
- **THEN** 用户可以切换到以下功能模块：
  - 数据导入
  - 智能填充
  - 配置管理
  - 历史记录
  - 设置

---

### Requirement: 数据导入界面
系统必须（SHALL）提供完整的数据导入功能界面。

#### Scenario: Word文件选择
- **GIVEN** 用户进入数据导入界面
- **WHEN** 用户点击"选择文件"按钮
- **THEN** 系统弹出文件选择对话框
- **AND** 过滤器仅显示.docx文件
- **AND** 选择后显示文件路径

#### Scenario: 表格预览和选择
- **GIVEN** 用户选择了Word文件
- **WHEN** 系统解析完成
- **THEN** 界面显示文档中所有表格的预览列表
- **AND** 每个表格显示：位置索引、行数、列数、表头预览
- **AND** 用户可以勾选需要导入的表格

#### Scenario: 客户和制程选择
- **GIVEN** 用户选择了要导入的表格
- **WHEN** 用户配置数据归属
- **THEN** 界面提供：
  - 客户下拉框（可新建）
  - 制程下拉框（可新建，根据客户筛选）
- **AND** 选择后显示当前选择

#### Scenario: 列映射配置
- **GIVEN** 用户需要指定列对应关系
- **WHEN** 界面显示列映射配置
- **THEN** 显示四个下拉框：
  - 项目列（显示表头名称供选择）
  - 规格列
  - 验收列
  - 备注列
- **AND** 默认按顺序自动映射

#### Scenario: 导入进度显示
- **GIVEN** 用户点击"开始导入"
- **WHEN** 导入进行中
- **THEN** 界面显示：
  - 进度条（显示百分比）
  - 当前处理的表格/行信息
  - 日志列表（滚动显示详细日志）
  - 取消按钮

### Requirement: 智能填充界面
系统必须（SHALL）提供完整的智能填充功能界面。

#### Scenario: 目标文件选择
- **GIVEN** 用户进入智能填充界面
- **WHEN** 用户选择目标Word文件
- **THEN** 界面显示文件信息
- **AND** 自动解析并显示表格列表

#### Scenario: 数据源选择
- **GIVEN** 用户选择了目标文件
- **WHEN** 用户配置数据源
- **THEN** 界面提供：
  - 客户选择
  - 制程选择
  - 显示该制程下的数据条数

#### Scenario: 匹配方式选择
- **GIVEN** 用户配置匹配参数
- **WHEN** 界面显示匹配选项
- **THEN** 用户可以选择：
  - 匹配方式（单选：相似度/Embedding/LLM+Embedding）
  - 相似度算法（仅相似度模式可选）
  - 匹配阈值（滑块或输入框，0-1）
  - 列权重配置

#### Scenario: 匹配预览表格
- **GIVEN** 用户点击"预览匹配结果"
- **WHEN** 系统完成匹配计算
- **THEN** 界面显示预览表格，列包含：
  - 行号
  - 原始项目
  - 原始规格
  - 匹配结果项目
  - 匹配结果规格
  - 将填充的验收内容
  - 将填充的备注内容
  - 匹配得分（颜色标识：绿色达标/红色未达标）
  - 操作（查看详情）

#### Scenario: 得分详情弹窗
- **GIVEN** 用户在预览表格中点击某行的"查看详情"
- **WHEN** 系统显示详情弹窗
- **THEN** 弹窗显示完整的得分计算详情：
  - 匹配方式
  - 使用的算法/模型
  - 项目列相似度及计算过程
  - 规格列相似度及计算过程
  - 权重配置
  - 综合得分公式和结果
  - 阈值对比
  - （LLM模式）匹配解释文本

#### Scenario: 确认填充
- **GIVEN** 用户查看预览结果满意
- **WHEN** 用户点击"确认填充"
- **THEN** 系统执行填充操作
- **AND** 显示填充进度
- **AND** 完成后显示"填充完成，共填充X行"
- **AND** 提供打开文件按钮

#### Scenario: 批量填充进度
- **GIVEN** 用户执行批量填充
- **WHEN** 填充进行中
- **THEN** 界面显示：
  - 总体进度条
  - 当前处理行
  - 已完成/总数
  - 取消按钮
  - 日志输出

### Requirement: 配置管理界面
系统必须（SHALL）提供配置管理的完整界面。

#### Scenario: 配置列表
- **GIVEN** 用户进入配置管理界面
- **WHEN** 界面加载完成
- **THEN** 显示已保存配置的列表，包含：
  - 配置名称
  - 创建时间
  - 最后修改时间
  - 匹配方式
  - 操作按钮（编辑/删除/导出）

#### Scenario: 配置编辑对话框
- **GIVEN** 用户点击"新建"或"编辑"
- **WHEN** 配置编辑对话框打开
- **THEN** 对话框包含标签页：
  - 基本信息（名称、描述）
  - 列映射配置
  - 匹配配置
  - AI参数配置
- **AND** 提供保存和取消按钮

#### Scenario: 配置导入导出
- **GIVEN** 用户在配置列表界面
- **WHEN** 用户点击导入/导出按钮
- **THEN** 导入：弹出文件选择对话框，选择.json文件
- **AND** 导出：弹出保存对话框，选择保存位置

### Requirement: 历史记录界面
系统必须（SHALL）提供操作历史查看和撤销界面。

#### Scenario: 历史记录列表
- **GIVEN** 用户进入历史记录界面
- **WHEN** 界面加载完成
- **THEN** 显示操作历史列表，包含：
  - 操作类型（导入/填充/删除）
  - 操作时间
  - 目标文件/客户制程
  - 操作详情摘要
  - 状态（可撤销/已撤销）
  - 操作按钮（查看详情/撤销）

#### Scenario: 历史详情查看
- **GIVEN** 用户点击某条历史的"查看详情"
- **WHEN** 详情弹窗打开
- **THEN** 显示完整的操作信息：
  - 操作时间
  - 操作类型
  - 影响的数据条数
  - 具体变更内容（JSON格式或表格形式）

#### Scenario: 撤销操作
- **GIVEN** 用户点击某条历史的"撤销"
- **WHEN** 系统执行撤销
- **THEN** 系统显示确认对话框"确定要撤销此操作吗？"
- **AND** 用户确认后执行撤销
- **AND** 更新历史状态为"已撤销"
- **AND** 显示"撤销成功"

### Requirement: 设置界面
系统必须（SHALL）提供全局设置界面。

#### Scenario: API Key配置
- **GIVEN** 用户进入设置界面
- **WHEN** 用户配置AI服务
- **THEN** 界面提供：
  - AI提供商选择
  - 对应的参数输入框（API Key、Endpoint等）
  - 测试连接按钮
  - 密码输入框（API Key不明文显示）

#### Scenario: 测试AI连接
- **GIVEN** 用户填写了AI配置
- **WHEN** 用户点击"测试连接"
- **THEN** 界面显示两个测试按钮：
  - "测试LLM连接" - 测试LLM服务
  - "测试Embedding连接" - 测试Embedding服务
  - "全部测试" - 同时测试两种服务
- **AND** 分别显示测试结果（成功/失败、响应时间）

#### Scenario: LLM连接测试结果
- **GIVEN** 用户点击"测试LLM连接"
- **WHEN** 测试完成
- **THEN** 成功显示"LLM连接成功 (响应时间: XXms)"（绿色）
- **AND** 失败显示具体错误信息（红色）

#### Scenario: Embedding连接测试结果
- **GIVEN** 用户点击"测试Embedding连接"
- **WHEN** 测试完成
- **THEN** 成功显示"Embedding连接成功 (维度: XXX, 响应时间: XXms)"（绿色）
- **AND** 失败显示具体错误信息（红色）

#### Scenario: 通用设置
- **GIVEN** 用户在设置界面
- **WHEN** 用户配置通用选项
- **THEN** 可配置项包括：
  - 配置文件存储路径
  - 数据库文件路径
  - 日志级别
  - 默认阈值

### Requirement: Prompt管理界面
系统必须（SHALL）提供Prompt模板管理界面，支持用户自定义LLM匹配的Prompt。

#### Scenario: Prompt管理入口
- **GIVEN** 用户在设置界面
- **WHEN** 用户点击"Prompt管理"选项
- **THEN** 系统打开Prompt管理界面

#### Scenario: Prompt编辑界面布局
- **GIVEN** 用户进入Prompt管理界面
- **WHEN** 界面加载完成
- **THEN** 界面显示以下区域：
  - 左侧：占位符列表和说明
  - 中间：Prompt编辑器（多行文本框）
  - 右侧：预览区域
  - 底部：保存/重置/取消按钮

#### Scenario: 编辑Prompt内容
- **GIVEN** 用户在Prompt编辑界面
- **WHEN** 用户在编辑器中输入或修改内容
- **THEN** 编辑器支持：
  - 多行文本编辑
  - 占位符语法高亮（如{{query_project}}显示为特殊颜色）
  - 实时字符数统计
  - 撤销/重做功能

#### Scenario: 插入占位符
- **GIVEN** 用户在Prompt编辑界面
- **WHEN** 用户在左侧占位符列表中点击某个占位符
- **THEN** 系统将该占位符插入到编辑器光标位置
- **AND** 占位符以特殊样式显示

#### Scenario: 预览Prompt效果
- **GIVEN** 用户编辑了Prompt内容
- **WHEN** 用户点击"预览"按钮
- **THEN** 右侧预览区域显示：
  - 使用示例数据填充后的完整Prompt
  - 预估Token数量
  - 格式检查结果

#### Scenario: 保存Prompt
- **GIVEN** 用户修改了Prompt内容
- **WHEN** 用户点击"保存"按钮
- **THEN** 系统验证Prompt格式
- **AND** 验证通过后保存到数据库
- **AND** 显示"保存成功"提示

#### Scenario: 重置为默认Prompt
- **GIVEN** 用户在Prompt编辑界面
- **WHEN** 用户点击"重置为默认"按钮
- **THEN** 系统弹出确认对话框
- **AND** 用户确认后恢复为系统默认Prompt
- **AND** 编辑器内容更新为默认模板

### Requirement: 状态栏和通知
系统必须（SHALL）在状态栏显示实时状态和提供通知功能。

#### Scenario: 状态栏显示
- **GIVEN** 应用程序运行中
- **WHEN** 执行各种操作
- **THEN** 状态栏实时显示：
  - 当前操作状态
  - 数据库连接状态
  - AI服务连接状态
  - 内存使用情况

#### Scenario: 操作完成通知
- **GIVEN** 长时间操作（如批量导入）完成
- **WHEN** 操作结束
- **THEN** 如果窗口不在前台，显示系统托盘通知
- **AND** 通知包含操作结果摘要

### Requirement: 错误处理和提示
系统必须（SHALL）提供友好的错误提示和处理机制。

#### Scenario: 文件访问错误
- **GIVEN** 用户选择的文件被其他程序占用
- **WHEN** 系统尝试访问文件
- **THEN** 显示友好错误提示"文件被占用，请关闭其他程序后重试"

#### Scenario: 网络错误
- **GIVEN** AI服务调用失败
- **WHEN** 系统捕获网络异常
- **THEN** 显示错误提示"网络连接失败，请检查网络设置"
- **AND** 提供重试按钮

#### Scenario: 未知错误
- **GIVEN** 发生未预期的异常
- **WHEN** 系统捕获异常
- **THEN** 显示通用错误提示"操作失败，请查看日志了解详情"
- **AND** 在日志文件中记录完整错误堆栈

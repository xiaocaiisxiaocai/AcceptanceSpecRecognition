# Word Processing Capability

## ADDED Requirements

### Requirement: Word文档读取
系统必须（SHALL）支持读取.docx格式的Word文档，并提取其中的所有表格信息。

#### Scenario: 读取包含单个表格的Word文档
- **GIVEN** 用户选择了一个包含单个表格的.docx文件
- **WHEN** 系统解析该文档
- **THEN** 系统返回表格列表，包含该表格的行数、列数和表头信息

#### Scenario: 读取包含多个表格的Word文档
- **GIVEN** 用户选择了一个包含多个表格的.docx文件
- **WHEN** 系统解析该文档
- **THEN** 系统返回所有表格的列表，每个表格包含位置索引、行数、列数和表头信息
- **AND** 用户可以选择需要处理的表格

#### Scenario: 读取不支持的文件格式
- **GIVEN** 用户选择了一个.doc格式的文件
- **WHEN** 系统尝试解析该文档
- **THEN** 系统显示错误提示"仅支持.docx格式"

---

### Requirement: 嵌套表格处理
系统必须（SHALL）支持识别和处理Word文档中的嵌套表格。

#### Scenario: 识别嵌套表格
- **GIVEN** Word文档中存在表格A，且表格A的某个单元格内包含表格B
- **WHEN** 系统解析该文档
- **THEN** 系统分别识别表格A和表格B
- **AND** 标记表格B为表格A的子表格
- **AND** 用户可以选择是否将嵌套表格作为独立数据源

#### Scenario: 提取嵌套表格数据
- **GIVEN** 用户选择了一个嵌套表格作为数据源
- **WHEN** 系统提取该表格数据
- **THEN** 系统正确提取嵌套表格中的所有行和列数据

---

### Requirement: 合并单元格处理
系统必须（SHALL）正确处理Word表格中的合并单元格。

#### Scenario: 导入时拆分合并单元格
- **GIVEN** Word表格中存在横向或纵向合并的单元格
- **WHEN** 系统提取表格数据用于存储
- **THEN** 系统将合并单元格拆分为多个独立单元格
- **AND** 每个拆分后的单元格填充相同的原始内容

#### Scenario: 填充时恢复合并单元格
- **GIVEN** 目标Word表格中存在合并的单元格
- **WHEN** 系统填充数据到该表格
- **THEN** 系统识别原始的合并结构
- **AND** 将数据填充到合并单元格中
- **AND** 保持原有的合并状态不变

---

### Requirement: 表格数据提取
系统必须（SHALL）能够按用户指定的列提取表格数据。

#### Scenario: 指定列提取
- **GIVEN** 用户选择了一个表格并指定提取第1、2、3、4列
- **WHEN** 系统提取数据
- **THEN** 系统返回指定列的所有数据行
- **AND** 第一行作为表头处理，不计入数据行

#### Scenario: 列索引越界
- **GIVEN** 用户指定提取的列索引超过表格实际列数
- **WHEN** 系统尝试提取数据
- **THEN** 系统显示错误提示"指定的列索引超出表格范围"

---

### Requirement: Word文档写入
系统必须（SHALL）支持向Word文档的表格中写入数据。

#### Scenario: 填充指定单元格
- **GIVEN** 用户指定了目标Word文档、表格索引、行号和列号
- **WHEN** 系统写入数据
- **THEN** 系统将数据填充到指定位置
- **AND** 保存修改后的文档

#### Scenario: 批量填充
- **GIVEN** 用户提供了多行待填充数据和对应的位置信息
- **WHEN** 系统执行批量写入
- **THEN** 系统按顺序填充所有数据
- **AND** 显示填充进度
- **AND** 完成后保存文档

#### Scenario: 填充过程中用户取消
- **GIVEN** 批量填充正在进行中
- **WHEN** 用户点击取消按钮
- **THEN** 系统停止填充操作
- **AND** 已填充的数据保留
- **AND** 提示用户"操作已取消，已完成X/Y项"

---

### Requirement: Word文件存储
系统必须（SHALL）将上传的Word源文件存储到数据库中，以便追溯数据来源。

#### Scenario: 存储源文件
- **GIVEN** 用户导入了一个Word文档
- **WHEN** 数据提取完成
- **THEN** 系统将Word文件二进制内容存储到数据库
- **AND** 记录文件名、文件哈希值和上传时间

#### Scenario: 检测重复文件
- **GIVEN** 用户尝试导入一个已存在的Word文档（相同哈希值）
- **WHEN** 系统检测到文件已存在
- **THEN** 系统提示"该文件已导入，是否覆盖或追加数据？"
- **AND** 用户可以选择覆盖、追加或取消

### Requirement: OK/NG格式转换
系统必须（SHALL）在填充【验收】列时支持OK/NG标记的格式标准化。

#### Scenario: 识别OK变体格式
- **GIVEN** 源数据【验收】列包含以下任意OK变体：
  - "OK"、"ok"、"Ok"、"oK"（大小写不敏感）
  - "OK✓"、"OK☑"、"OK⬛"、"OK■"、"OK√"
  - "✓OK"、"☑OK"
- **WHEN** 系统读取并准备填充数据
- **THEN** 系统识别这些为OK标记

#### Scenario: 识别NG变体格式
- **GIVEN** 源数据【验收】列包含以下任意NG变体：
  - "NG"、"ng"、"Ng"、"nG"（大小写不敏感）
  - "NG✗"、"NG☒"、"NG⬛"、"NG■"、"NG×"
  - "✗NG"、"☒NG"
- **WHEN** 系统读取并准备填充数据
- **THEN** 系统识别这些为NG标记

#### Scenario: 转换为标准OK格式
- **GIVEN** 用户启用OK/NG格式转换
- **AND** 源数据为"OK☑ 已通过检验"
- **WHEN** 系统填充【验收】列
- **THEN** 系统将"OK☑"转换为配置的标准格式（默认"OK"）
- **AND** 保留附加文字"已通过检验"
- **AND** 最终填充内容为"OK 已通过检验"

#### Scenario: 转换为标准NG格式
- **GIVEN** 用户启用OK/NG格式转换
- **AND** 源数据为"NG⬛️ 尺寸超差0.5mm"
- **WHEN** 系统填充【验收】列
- **THEN** 系统将"NG⬛️"转换为配置的标准格式（默认"NG"）
- **AND** 保留附加文字"尺寸超差0.5mm"
- **AND** 最终填充内容为"NG 尺寸超差0.5mm"

#### Scenario: 保留纯文字内容
- **GIVEN** 源数据【验收】列为"符合IEC标准"（不含OK/NG标记）
- **WHEN** 系统填充【验收】列
- **THEN** 系统直接填充原始内容
- **AND** 不进行OK/NG转换

#### Scenario: 仅OK/NG无附加文字
- **GIVEN** 源数据【验收】列仅为"OK⬛"
- **WHEN** 系统填充【验收】列
- **THEN** 系统转换为标准格式"OK"
- **AND** 不添加额外空格或文字

#### Scenario: 禁用OK/NG格式转换
- **GIVEN** 用户禁用OK/NG格式转换功能
- **WHEN** 系统填充【验收】列
- **THEN** 系统保留原始格式
- **AND** 直接填充"OK☑ 已通过检验"

### Requirement: 关键字高亮填充
系统必须（SHALL）在填充Word文档时对配置的关键字进行高亮显示。

#### Scenario: 单个关键字高亮
- **GIVEN** 用户启用关键字高亮功能
- **AND** 配置关键字"危险"，高亮颜色为黄色
- **AND** 填充内容为"注意：此区域存在危险气体"
- **WHEN** 系统填充到Word文档
- **THEN** 系统在Word文档中将"危险"设置为黄色高亮
- **AND** 其他文字保持正常格式

#### Scenario: 多个关键字高亮
- **GIVEN** 用户配置关键字"危险"、"禁止"、"注意"
- **AND** 填充内容为"注意：禁止触摸，有危险"
- **WHEN** 系统填充到Word文档
- **THEN** 系统分别高亮"注意"、"禁止"、"危险"三个词
- **AND** 所有关键字使用相同的高亮颜色

#### Scenario: 关键字不区分大小写
- **GIVEN** 配置关键字"WARNING"
- **AND** 填充内容为"warning: High Temperature"
- **WHEN** 系统填充到Word文档
- **THEN** 系统高亮"warning"（匹配不区分大小写）

#### Scenario: 关键字在备注列高亮
- **GIVEN** 用户启用关键字高亮
- **AND** 配置关键字"复检"
- **WHEN** 系统填充【备注】列内容"需要复检确认"
- **THEN** 系统高亮【备注】列中的"复检"

#### Scenario: 禁用关键字高亮
- **GIVEN** 用户禁用关键字高亮功能
- **WHEN** 系统填充Word文档
- **THEN** 系统不进行关键字检测和高亮
- **AND** 直接填充纯文本

#### Scenario: 高亮颜色配置
- **GIVEN** 用户配置高亮颜色为红色(#FF0000)
- **WHEN** 系统填充包含关键字的内容
- **THEN** 系统使用红色高亮显示关键字

---

### Requirement: 填充格式保持
系统必须（SHALL）在填充时保持Word文档的原有格式。

#### Scenario: 保持单元格字体格式
- **GIVEN** 目标Word表格单元格原有字体为"微软雅黑 10号"
- **WHEN** 系统填充数据到该单元格
- **THEN** 填充的文本继承原有字体设置
- **AND** 除关键字高亮外不改变其他格式

#### Scenario: 保持单元格对齐方式
- **GIVEN** 目标Word表格单元格设置为居中对齐
- **WHEN** 系统填充数据
- **THEN** 填充后保持居中对齐

#### Scenario: 关键字高亮不影响其他格式
- **GIVEN** 填充内容需要高亮关键字
- **WHEN** 系统应用高亮
- **THEN** 仅添加高亮背景色
- **AND** 保持字体、字号、颜色等其他格式不变

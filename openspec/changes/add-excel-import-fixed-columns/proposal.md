# Change: 增加 Excel（.xlsx）上传并按“列序号”导入数据库

## Why
当前系统仅支持从 Word（.docx）表格导入验收规格数据。实际项目中，验收数据也常以 Excel（.xlsx）维护与流转。如果缺少 Excel 导入能力，会导致：
- 需要先手工转换为 Word 或重复录入，成本高、易出错
- 同一套验收数据难以在不同来源（Word/Excel）之间统一沉淀与复用

Excel 的典型复杂度包含：工作表不需要导入、表头跨行、起始列不是第 1 列、合并单元格、公式/数值/日期等混合类型。需要在导入前提供可预览、可配置、可校验的导入流程。

## What Changes
- 新增 Excel（.xlsx）文件上传，并解析工作簿工作表列表（名称、索引、可见性、已用区域等）。
- 导入流程新增“按工作表配置导入”：
  - 选择需要导入的工作表（支持批量导入多个工作表；可跳过不需要的页）。
  - **不复用“列映射规则”功能**；Excel 导入采用“明确指定第几列是什么字段”的方式配置：
    - 项目列（必填）
    - 规格内容列（必填）
    - 验收标准列（可选）
    - 备注列（可选）
  - 支持表头跨行与数据起始行配置（例如表头占 2~3 行、数据从第 N 行开始）。
  - 合并单元格展开（将合并区域的值填充到区域内所有单元格）以保证行级提取稳定。
- 导入时复用现有数据模型 `AcceptanceSpec`（项目/规格/验收/备注 + 客户/制程归属），并记录来源文件与导入时间，提供逐表逐行错误汇总。

## Non-Goals（本次不做）
- 不做“按表头文字自动识别列”（避免与现有列映射规则能力重叠）。
- 不做“在一个工作表中自动识别多块表格区域”（若需要，将通过显式区域配置作为后续扩展）。
- 不做“将 Excel 导入后自动写回 Excel”（本系统当前写回能力以 Word 为主）。
- 不做“去重/幂等”（同一文件或同一工作表重复导入可能产生重复数据）。

## Impact
- Affected specs（planned）：
  - `user-interface`（导入界面支持 Excel 与按工作表配置）
  - `data-storage`（来源文件需要支持 Excel 或扩展文件类型）
  - 新增 `excel-processing`（Excel 解析/合并单元格/行提取/校验）
- Affected code（planned）：
  - API：文件上传与 Excel 解析预览、导入接口
  - Core：Excel 解析器（工作表枚举、单元格读取、合并单元格展开）
  - Web：导入向导增加 Excel 入口与每工作表配置表单

## Open Questions
已确认口径（本变更按此实现）：
1. 支持的格式范围：**只支持 `.xlsx`**。
2. 列号基准：**1-based（第1列=A）**。
3. “一张表里多块区域”：**不考虑**（默认每个工作表只有一个连续的表格区域）。
4. 空行与结束条件：**读到已用区域末尾**（不做“连续空行提前停止”）。
5. 合并单元格策略：**表头+数据区都展开**。
6. 公式单元格策略：**不考虑公式**（按普通值读取；不做公式计算/缓存结果优先等策略）。

后续可能形成单独变更（本次不涉及）：
- 去重/幂等：按（客户/制程/项目/规格）做去重策略（跳过/覆盖/报错）。
